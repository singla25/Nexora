# Nexora - Modular Social Networking Platform for WordPress Nexora is a custom WordPress plugin that transforms WordPress into a modular, extensible social networking platform. It combines WordPress core capabilities with custom-built systems for authentication, dashboards, chat, notifications, and marketplace integration. The system is designed with a hybrid architecture using WordPress CPTs for structured data and custom database tables for high-performance features like chat and notifications. --- ## 🚀 Overview Nexora provides: * Custom authentication system * Multi-profile dashboard (User & Vendor) * Connection management system * Real-time chat (AJAX-based) * Notification system (custom DB) * Modular dashboard tabs * Extensible marketplace integration * Admin management panel --- ## 📦 Plugin Structure Main plugin entry: ``` wp-content/plugins/nexora/nexora.php ``` Core modules: * Authentication (`login-signup/`) * Dashboard (`dashboard/`) * Chat (`chat/`) * Notifications (`includes/`) * Admin (`admin/`) --- ## ⚙️ Installation 1. Clone repository: ``` git clone https://github.com/singla25/Nexora.git ``` 2. Place plugin: ``` wp-content/plugins/nexora/ ``` 3. Activate via WordPress Admin 4. Create required pages: ``` [profile_login] [profile_registration] [vendor_registration] [nexora_dashboard] ``` 5. Flush permalinks: ``` Settings → Permalinks → Save Changes ``` --- ## 🔐 Authentication System Located in: ``` login-signup/ ``` Features: * Custom login & registration * Vendor registration * Auto-login after signup * OTP-based password recovery * Google reCAPTCHA support Creates: * WordPress users * Profile CPTs (`user_profile`, `vendor_profile`) * User meta linking profiles --- ## 👤 Profile System ### Profile Types | Role | CPT | Profile Type | | ---------- | -------------- | ------------ | | subscriber | user_profile | user | | vendor | vendor_profile | vendor | Mapping handled via: ``` NEXORA_DASHBOARD_HELPER::ROLE_MAP ``` --- ## 🧠 Dashboard System Location: ``` dashboard/ ``` ### Routing ``` /dashboard/{username} ``` ### Context Types * guest * owner * viewer This allows dynamic UI rendering based on access level. --- ## 📑 Dashboard Tabs | Tab | Visibility | | ------------- | ---------- | | Information | All | | Connections | Users only | | Content | Owner only | | Marketplace | Owner only | | Notifications | Owner only | --- ## 🧾 Profile Information Sections User: * Personal * Address * Work * Documents * Security Vendor: * Personal * Address * Business * Documents * Security Vendor-specific fields include: * Business name, type, email * GST, category, service areas * Website, years in business --- ## 🔗 Connections System Stored in: ``` user_connections (CPT) ``` Features: * Send / accept / reject requests * Remove connections * Mutual connections * Connection history Statuses: * pending * accepted * rejected * removed --- ## 🔔 Notification System Custom table: ``` wp_nexora_notifications ``` Tracks: * Actor & receiver * Message * Type * Read status * Timestamp Used for: * Connection updates * Dashboard alerts * Admin monitoring --- ## 💬 Chat System Located in: ``` chat/ ``` Custom tables: * threads * participants * messages * message_meta Features: * One-to-one chat * Connection-based messaging * Thread-based conversations * AJAX polling * Admin read-only view Security: * Access validation * Participant checks * Message sanitization --- ## 🧩 Marketplace (Extension Ready) Marketplace tab exists inside dashboard but is modular. Template: ``` dashboard/templates/tabs/market.php ``` Inject content via: ```php add_filter('nexora_marketplace_content', function($html, $ctx) { return do_shortcode('[your_marketplace_shortcode]'); }, 10, 2); ``` Status: * UI scaffold ready * Role-based visibility implemented * Awaiting marketplace module integration --- ## 🛠 Admin Module Location: ``` admin/ ``` Features: * CPT registration * Metabox management * Plugin settings * Notification viewer * Chat viewer Settings include: * Default images * Email config * reCAPTCHA keys --- ## 🧱 Data Architecture ### WordPress CPTs * user_profile * vendor_profile * user_connections * user_content ### Custom Tables * notifications * chat (threads, messages) Hybrid design ensures: * Flexibility (CPT) * Performance (custom tables) --- ## 🔒 Security Highlights * Nonce validation in AJAX * Role-based access control * Sanitization & escaping * Chat access restrictions * Admin-only controls --- ## 🌐 Frontend Assets Global: ``` assets/css/style.css assets/js/script.js ``` Dashboard: ``` dashboard/assets/ ``` Chat: ``` chat/assets/ ``` Authentication: ``` login-signup/assets/ ``` --- ## 🔁 Routing & Access Control * Admin access restricted to admins * Non-admins redirected to dashboard * Custom login replaces wp-login * Admin bar hidden for non-admin users --- ## 🔌 Extensibility ### Add New Dashboard Role 1. Register WP role 2. Create profile CPT 3. Map role 4. Define tab visibility 5. Add templates --- ### Add Dashboard Tab 1. Register tab label 2. Define visibility 3. Add template file --- ### Extend Marketplace Use: ```php nexora_marketplace_content ``` --- ## 🧪 Testing Current approach: * PHP lint checks * JS syntax checks * Manual AJAX validation Future improvements: * PHPUnit integration * WP CLI testing * Static analysis tools --- ## 🛣 Roadmap * Marketplace module integration * Improved OTP system * Settings sanitization improvements * Automated testing setup * WebSocket-based chat (future) --- ## 🧰 Tech Stack * WordPress * PHP (OOP) * MySQL * AJAX * jQuery * SweetAlert2 * Google reCAPTCHA --- ## 📌 Summary Nexora is a modular WordPress-based social platform built with scalability and separation of concerns in mind.
